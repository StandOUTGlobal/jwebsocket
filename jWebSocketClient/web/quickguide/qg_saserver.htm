jWebSocket Stand-Alone Server

This section describes how to use the jWebSocket Stand-Alone Server .jar on the various operating systems including Windows, Unix and Linux. This topic will be interessting for you when you are a developer and intend to develop new client software based upon the jWebSocket Server or if you are a tester. The jWebSocketSAServer.jar which includes all required libraries. The benefit is that it easily can be started from the shell or command line window without any installation or special configuration.

Like all normal desktop applications the jWebSocket Stand-Alone Server is terminated when you log off the system. Thus for production systems it is recommended to use either the jWebSocket Service (for Windows) or the jWebSocket Web Application (for all operating systems).

Prerequisities on the server

The jWebSocket server is based on pure Java technology. For your server please ensure that you have at least the Java Runtime Environment (JRE) 1.5 installed. If not you can get it here . For Windows we recommend to put the path to java.exe into your PATH enviroment variable. If not, you may need to adjust the provided start batch.

Download, Install and Run the jWebSocket Server

To set up the jWebSocket server on your MS Windows system please....

  1. Create a folder of your choice (e.g. c:\program files\jWebSocket).
  2. Download the jWebSocketSAServer.zip which includes the jWebSocketSAServer.jar, all required libs and jWebSocketSAServer.bat from the download section to this folder.
  3. For Windows environments please start the batch either from the command shell or the Windows Explorer.
    If the path to the java.exe is not part of your PATH environment variable, please adjust the jWebSocketSAServer.bat batch accordingly (see inline documentation in the batch).
  4. For Unix and Linux environments please type java -jar jWebSocketSAServer.jar in the command shell.

Command line arguments

These are the command line arguments interpreted by the main class of the jWebSocketSAServer.jar.

  • port=[port], default 8787
  • prot=[json|csv|xml|custom], default=json
  • loglevel=[all|debug|info|error|warn]
  • sessiontimeout=[timeout in ms], default 120000ms = 2 min

jWebSocketServer.bat


For MS-Windows environments the jWebSocketServer.zip from the download area includes batch file jWebSocketServer.bat.

java -jar jWebSocket.jar %1 %2 %3 %4 %5 %6

From the windows command line your can call jWebSocketServer.bat with the above mentioned optional arguments.
