jWebSocket Windows Service

This section describes how to set up the jWebSocket server so that it runs as a windows service, e.g. on a Windows server. This topic will be interessting for you when you intend to run the jWebSocket Server on a dedicated production system.

It is quite simple to run the jWebSocketSAServer.exe as a stand-alone server by simply double-clicking it or putting a short-cut to it into the windows auto-start group. However, the drawback is that normal desktop applications are terminated when you log off the system. Unlike usual applications services remain running in the background even when you log off. Thus on a stand-alone server you need to remain logged in to run the jWebSocket Server which is not recommended due to security reasons.

To run the jWebSocket server w/o being logged in you either can run it as a Windows service by using the jWebSocketService.exe (described below) or alternatively embed it into a web application which is described here.

Installing/uninstalling jWebSocketService

To run the jWebSocket server as a Windows service please follow the following steps.

  • Download the jWebSocketService.zip from the download area (you also can click here).
  • Unpack the .zip file into a local folder of your choice (e.g. c:\program files\jWebSocket\service)
  • Change to that folder and run jWebSocketInstallService.bat to register the jWebSocketService.exe as an Windows service.
  • Check if the service has been properly installed in the Windows Service Manager (select Start/Administrative Tools/Services).
  • Right-click the jWebSocket Service entry and select Start.
  • The service starts and the jWebSocket server is available.
  • When you re-start the system the service will be started automatically.
