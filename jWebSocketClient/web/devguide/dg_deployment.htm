jWebSocket Deployment Process

This section describes how to deploy the jWebSocket application to a stage or production system. For the required tools please refer to the Prerequisities page.

The Java Docs are auto-created based upon the inline-documentation in the jWebSocketClient/javadocs folder.

The files for the Quick Guide are maintained in the jWebSocketClient/quickguide folder.

The files for the Developer Guide are maintained in the jWebSocketClient/devguide folder.

Process

To create a full deployment please follow the following steps:

  1. Minify/Obfuscate JavaScript and CSS files with Jasob Obfuscator.
  2. Update version number in JWebSocketConstants.java
  3. Update all version numbers in pom.xml files to current version
  4. Update version number in context.xml for web app (we currently simply use NotePad++ for file search and replace)
  5. Run "Generate Javadoc" for each jWebSocket module within the project
    we have a createJavaDocs.bat now in the deployment folder
    (this creates all required Java documentation files in the jWebSocketClient\web\javadocs folder)
  6. Open IDE and "Clean And Build" the jWebSocket-Parent project
    (this creates all required .jar files of the modules in the jWebSocketExecutables folder)
  7. Generate the jWebSocket Server Executables by running Exe4J with createRuntimeFiles.bat.
    (this creates the jWebSocketServer<32|64>-<ver>.exe and jWebSocketService<32|64>-<ver>.exe in the jWebSocketRTE/bin folder)
  8. Update Version numbers in all batches the jWebSocketRTE/bin folder.
  9. Update the createDownloadFiles.bat in the jWebSocketDeployment folder to the new version number and run it.
    (this creates all .zip files for the download area in jWebSocketDeployment)
  10. Check correct file sizes on download page (compare with last recently created files)
  11. Update jwsClientToWebsite.bat to new version number, adjust it to new files if needed and run it.
  12. Run createJWebSocketWeb.bat to create jWebSocket online web.
  13. Create announcement in blog
  14. Create announcement in twitter

Change all version numbers in

  4. Check status of native WebSocket support in
