jWebSocket Permissions and Access Control

The access to certain resources or functions of the jWebSocket server is controlled by roles and rights. Both rights and roles are specified in the jWebSocket.xml. A user is always assigned to one or more roles and each role covers one or more rights. Thus the user's effective rights are calculated by all rights of those roles the user is assigned to.

Rights

Rights are specified by an id, a namespace and a description in a <rights> section of the jWebSocket.xml config file. Rights are not directly assigned to a certain user but only to roles (see below).

<rights>

  <right>
    <id>right0</id>
    <ns>org.jWebSocket.global</ns>
    <description>Description for right0</description>
  </right>
  <!-- insert more rights here -->
</rights>

Rights can either be global or plug-in specific. Hence there can be multiple <rights> sections in a config file, one global one and optionally one per plug-in if the plug-in needs a authorization mechanism. Because plug-ins already have a namespace the namespace for the plug-in specific right can be inherited and does not explicitely be specified. If it is specified it overwrites the inherited value. Internally all effective rights of a use are maintained in a single map with <namespace>.<id> acting as the key and the Right class as value.

Roles

Like the Rights roles also are specified by an id, a namespace and a description in a <rights> section of the jWebSocket.xml config file. A user is assigned to one or more roles. His effective rights are are calculated by all rights of the roles he is assigned to.

<roles>
  <role>
    <id>Administrator</id>
    <description>Administrators</description>
    <rights>
      <right>org.jWebSocket.plugins.rpc.rpc</right>
      <right>org.jWebSocket.plugins.rpc.rrpc</right>
    </rights>
  </role>

  :
</roles>
 

Unlike the rights the roles do not have an namespace but a list of rights. The <rights> section within a role specifies which rights a user obtains when he is assigned to that role.

Users

In the users sections of the jWebSocket.xml all users that may login at the jWebSocket server are listed. A user is specified by its loginname, firstname, lastname and a description. The status field allows the block a user. In the roles section it is specified to which roles a user is assigned to.

<users>
  <user>
    <loginname>aschulze</loginname>
    <firstname>Alexander</firstname>
    <lastname>Schulze</lastname>
    <password>t0p-s3cr3t</password>
    <description>jWebSocket Founder</description>
    <!-- 0=blocked 1=allowed ... future states pending -->
    <status>1</status>
    <roles>
      <!-- the rights of all roles are merged -->
      <role>Administrator</role>
      <role>User</role>
      <role>Guest</role>
    </roles>
  </user>
<user>
 

Please notice that the password field is stored in plain text here, so take care that the jWebSocket.xml file is not accessible by unauthorized people on the network.
