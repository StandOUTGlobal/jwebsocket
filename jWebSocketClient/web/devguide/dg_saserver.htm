jWebSock Server as Stand-Alone Server

This Chapter describes how to run the jWebSocket Server as a Stand-Alone Server.
This topic will be interessting for you when you intend to run the jWebSocket Server on a development or test system.

Running jWebSocketServer.jar as stand-alone application

The jWebSocketServer.jar together with the libraries in the /lib path contains all required Java classes to run the jWebSocket Server on a stand-alone system.

You can run multiple instances of the jWebSocket stand-alone server on a single system provided that you use different listener ports.

Command line arguments

These are the command line arguments interpreted by the main class of the jWebSocketServer.jar.

  port=[port], default 8787
  prot=[token|plain], default=token
  loglevel=[all|debug|info|error|warn]
  sessiontimeout=[timeout in ms], default 120000ms = 2 min

 


jWebSocketServer.bat


For MS-Windows environments the jWebSocketServer.zip from 
the download area includes batch file jWebSocketServer.bat.


java -jar jWebSocket.jar %1 %2 %3 %4 %5 %6

		
From the windows command line your can call jWebSocketServer.bat with 
		the above mentioned optional arguments.
