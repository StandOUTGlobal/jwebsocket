Server-Plug-Ins

The jWebSocket server itself only provides fundamental communication capabilities. All higher lever functions are implemented in plug-ins. Multiple plug-ins can be ordered in a chain. This concept allows you to easily add custom specific WebSocket services according to your needs based on what already exists. The included jWebSocket SystemPlugIn demonstrates how to implement own plug-ins and it already provides a couple of nice-to-have functions for your WebSocket work. We will extend the System plug-in on demand or provide further plug-ins like the Remote Procedure Call (RPC) plug-in or e.g. for the future clustering options.

A server plug-in basically has three methods:

Method Description
connectorStarted

Called from the plug-in chain when a new client connects to the jWebSocket server.
If a client for instance should be automatically registered at a certain data stream this would be the right place to do this.
processData

Called from the plug-in chain every time when a client sends data to the server.
The data is passed to method as object. It can be casted according to whatever the underlying protocol requires.
connectorTerminated

Called from the plug-in chain when a client was disconnected from the jWebSocket server.
If a client for instance is registered at a data stream this is the right place to unregister its connector from these stream.
